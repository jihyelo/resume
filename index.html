사용 기술

유효성 검사

• 이용약관 동의 페이지에서 전체동의와 개별동의 체크박스의 상태를 관리하고, 사용자가 선택한 상태에 따라 버튼의 활성화 결정합니다.

• React-hook-form을 활용하여 회원가입 폼을 개발하고 유효성 검사를 통해 데이터의 정확성을 보장하였습니다.

• 실시간으로 오류 메시지를 표시하여 사용자들이 입력을 수정할 수 있도록 안내하고, 사용자 친화적인 인터페이스를 제공하였습니다.

게시판

• 굿즈와 공구 페이지 간의 토글 기능을 구현하고, Next.js의 동적 라우팅을 활용하여 각 페이지별로 필요한 데이터를 요청하고 렌더링하는 방식으로 카드를 동적으로 로드했습니다.

• Recoil을 활용하여 전역 상태로 검색어를 관리하고, 이를 이용하여 제목과 아티스트를 검색하는 기능을 구현했습니다. 검색어가 변경될 때마다 카드를 필터링하여 해당하는 결과만 보여주도록 하였습니다.

• 사용자가 선택한 정렬 옵션에 따라 카드를 정렬하는 기능을 구현했습니다. 이를 위해 React Query를 사용하여 서버에서 데이터를 요청하고, 정렬된 결과를 렌더링했습니다.

• react-js-pagination 라이브러리를 사용하여 페이지네이션을 구현했습니다.

• 로그인할 때 서버로부터 해당 사용자의 구독 정보를 가져와서 Recoil 상태나 컨텍스트에 저장하고, 해당 정보를 이용하여 서버로부터 받은 데이터를 필터링하여 사용자가 구독한 아티스트만 보여줍니다.

상세페이지

• 상품 정보를 데이터베이스에서 불러와 사용자에게 보여주고, ReactSelect을 활용하여 옵션 선택 기능을 구현했습니다. 또한 Recoil을 사용하여 전역 상태를 관리하여 옵션 선택에 따른 가격 및 수량을 자동으로 계산하였습니다.

• 구매자 정보를 입력할 때 우편번호를 쉽게 찾을 수 있도록 다음의 오픈 API를 활용하여 우편번호 검색 기능을 제공했습니다.

• 모든 필수 입력 항목 및 동의 사항에 대한 체크가 완료되면 버튼이 활성화되고, 버튼을 누르면 클라이언트에서 서버로 구매자의 정보를 전송합니다. 서버는 해당 정보를 받아서 데이터베이스에 저장하고 사용자가 구매금액을 입금을 할 수 있도록 판매자의 계좌 정보가 적힌 모달 창을 보여줍니다.

글쓰기

• 제목과 내용을 쓰고나면 해시태그, 상품 옵션, 배송 옵션과 같은 세부 정보를 추가하거나 삭제할 수 있도록 하고, 이를 통해 Recoil 상태를 업데이트합니다.

• 마지막 단계에서는 미리보기 화면을 제공하여 사용자가 입력한 정보를 최종적으로 확인할 수 있도록 하였습니다. 사용자가 작성한 글이 어떻게 보일지 미리 확인하고 수정할 기회를 제공함으로써 실수를 줄이고 정확한 정보를 입력할 수 있습니다.

• 글 작성을 완료하기 전까지는 Recoil 상태에 저장된 정보를 계속 유지합니다. 이를 통해 사용자가 step을 이동하거나 페이지를 새로고침해도 작성 중인 정보가 사라지지 않습니다. 이러한 방식으로 글 작성 중에도 안정적으로 상태를 유지할 수 있으며, 사용자 경험을 향상시킬 수 있습니다.

• 사용자가 글 작성을 완료하면, 해당 데이터가 서버로 전송되어 데이터베이스에 저장되고 이후에 굿즈/공구 게시판에서 확인할 수 있습니다.

기획, 메인페이지, 카테고리페이지, 상세페이지 UI와 기능 구현 및 API연동

① 메인페이지

- 서버에서 카테고리별 상품 데이터를 불러오는 API를 호출하고 이 때, 카테고리 ID를 파라미터로 전달하여 해당 카테고리에 속하는 상품들을 가져옴
- API 응답으로 받은 데이터를 클라이언트 측에서 처리하여 데이터를 동적으로 렌더링
- React-slick 라이브러리를 사용하여 캐러셀 구현
- Sorting(날짜/예매율)

② 카테고리페이지

- 사용자가 카테고리를 선택하면 URL에 해당 카테고리 정보를 쿼리스트링으로 추가하여, 웹페이지를 로드할 때 선택된 카테고리를 확인하고 그에 맞는 데이터를 서버에 요청

③ 상세페이지

- 상세 페이지로 이동할 때, 해당 상품의 고유한 식별자(ID)를 사용하여 서버에 요청
- React Calendar 라이브러리 이용하여 예매 가능한 날짜를 표시하고, 사용자의 예약 선택 처리하는 예매달력 구현
- 사용자가 예약을 진행할 때, 해당 사용자의 인증 토큰을 확인하여 인증된 사용자인지 확인
- 로그인 된 사용자가 아니라면, 로그인 페이지로 이동
- 로그인 된 사용자라면, 해당 예약 정보를 서버에 전송하여 예약을 완료

담당 업무

사이즈선택, 즉시구매/즉시판매, 구매입찰/판매입찰, 결제페이지

① 사이즈선택

- 원하는 상품을 클릭하면 사이즈를 선택하는 화면으로 이동 하며 동적 라우팅으로 상품 정보를 불러옴
- 쿼리스트링을 이용하여 선택한 데이터를 다음 페이지(즉시/입찰)로 전달하여 통신 과정을 줄임

② 즉시구매/즉시판매

- 해당 상품의 사이즈를 선택하면, 처음 받아온 데이터를 제품의 판매 입찰과 즉시 판매 토글에 따라 데이터를 다르게 출력
- 즉시구매/즉시 판매에서 서버에서 받아온 상품데이터의 가격을 저장하여 즉시구매하기 버튼을 누를 때 페이지 이동과 함께 가격데이터를 전송

③ 구매입찰/판매입찰

- 즉시구매/즉시판매와는 다르게 구매입찰/판매입찰에서는 유저가 직접 입력한 금액을 입력
- 입찰하기 버튼을 누를 때 페이지 이동과 함께 가격데이터를 전송
- 입찰 마감기한을 선택하면, 날짜가 자동계산되어 출력

④ 결제페이지

- 토큰으로 고객 정보를 확인, 해당 고객이 보유한 포인트를 사용하여 결제
- 구매/판매에 필요한 결제 금액을 표시하고 내가 입력한 포인트를 실시간으로 출력하여, 포인트와 상품 또는 배송비의 총액이 일치하는지를 확인 가능
- 보유 포인트가 입력 포인트보다 클 경우 경고메시지 출력하여, 보유한 포인트 내에서 올바른 입력을 유도
- 입력 금액과 결제가 필요한 총액이 일치할 경우 결제 버튼 활성화하여, 불일치 시 다음 과정으로 넘어가면서 결제 재선택을 요구하는 불필요한 과정을 축소
- 결제된 금액(포인트)를 서버로 전달하여 사용한 포인트를 보유 포인트에서 차감
